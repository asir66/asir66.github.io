MSF

Metasploit，全称为 The Metasploit Framework，简称MSF。MSF 高度模块化，即框架由多个 module 组成，是全球最受欢迎的渗透测试工具之一

简单说来就是大型模板+脚本，方便渗透测试

Kali 系统 metasploit 框架目录路径: /usr/share/metasploit-framework

  1. data：包含metasploit用于存储某些漏洞、单词列表、图像等所需二进制文件的可编辑文件。
  2. documentation：包含框架的可用文档。
  3. lib：metasploit的库文件夹。
  4. plugins：用来存放metasploit的插件。
  5. scripts：用来存放metasploit的脚本，包括meterpreter及其它脚本。
  6. tools：存放多种的命令行实用程序。
  7. modules：存储metasploit的模块文件。

Metasploit模块简介

Metasploit-framework 中分别有以下几种类型的模块：

模块名 模块功能 模块介绍
auxiliary 辅助模块 辅助渗透过程，例如主机发现，端口扫描，漏洞检测等
exploits 漏洞利用模块 包含msf中所有的漏洞利用脚本，各模块按照漏洞影响的系统和交互方式，有着固定的存放路径。命名规则：操作系统/协议类型
payloads 攻击载荷 包含漏洞利用成功后在目标机器上执行的代码。包括反向shell、正向shell、meterpreter（msf中内置的远控木马）
post 后渗透模块 包含漏洞利用成功获得目标系统访问权限后要做的操作，例如搜集敏感信息、提升权限等
encoders 编码器模块 主要包含各种编码工具，对payload进行编码加密，已绕过入侵检测和过滤系统
evasion 免杀模块 用来生成可以绕过反病毒软件检测的免杀payload
nops 空指令模块 生成各种架构系统的空操作指令，用来提高payload的稳定性及扩充长度

metasploit是一个分模块的大型框架，这样渗透的时候就会很有逻辑性，实用。

msfvenom

MSF的一个独立的payload生成器，可以生成在目标机上直接运行的木马

选项很多，重点两个：-p -f

MSF创建监听器

可以直接用handler模块来开启监听

会话详情可以实用session指令查看
msf6 > use exploit/multi/handler

Metasploit的Payload类型

  • Stageless payload
    Stageless payload: <platform>/[arch]/<single>

Stageless payload 是一个二进制文件，包含Meterpreter的所有必需部分以及所有必需的扩展全部捆绑在 一起，将完整的payload都编译在木马中，体积庞大，可直接独立地植入目标系统进行执行

  • Staged payload
    Staged payload: <platform>/[arch]/<stage>/<stager>

Staged payload 负责建立目标用户与攻击者之间的网络连接，将执行传递到另一个阶段，MSF提供了传输器和传输体配对分阶段植入的技术，由渗透攻击模块首先植入代码短小精悍且可靠的传输器载荷，然后在运行传输器载荷时进一步下载传输体载荷并执行，比如加载meterpreter、VNC桌面控制等复杂的大型攻击载荷。

Meterpreter

Meterpreter是一个高级的、动态的、可扩展的payload，简单理解是一个高级的cmd，里面封装了Metasploit的功能

Meterpreter是Metasploit框架中的一个扩展模块，作为溢出成功以后的攻击载荷使用，攻击载荷在溢出攻击成功以后给我们返回一个控制通道。使用它作为攻击载荷能够获得目标系统的一个Meterpreter shell的链接。

Meterpreter shell 作为渗透模块有很多有用的功能，比如添加一个用户、隐藏一些东西、打开shell、得到用户密码、上传下载远程主机的文件、运行cmd.exe、捕捉屏幕、得到远程控制权、捕获按键信息、清除应用程序、显示远程主机的系统信息、显示远程机器的网络接口和IP地址等信息。

  6. 使用thinkphp5-rce漏洞利用模块攻击靶机2

我对实验的总结：

  1. 对search命令的使用

  2. 对msf框架中模块与模块的了解

  3. 对端口扫描方法的掌握

  4. 对使用msf的一般操作过程

实验报告

由于这个实验做的过程中出现了些许小问题，并由实验本身是并不太难的。我先把实验的命令行放在前面，后面再说遇到的问题

实验步骤，

遇到的问题：

  1. tmp.exe位置不对，报错： Post failed: Msf::OptionValidateError One or more options failed to validate: REXEPATH.
    原因，因为我打开的命令行是直接打开，而不是从root处打开，所以我的tmp.exe是放在根目录下的

  2. 攻击机ip不对：Handler failed to bind to 10.1.1.91:4444:- -，搜索之后说是ip不对，ifconfig之后发现应该是对的，改了个10.1.1.255，可以了，重启也行。大概是端口冲了吧

  3. 到最后执行完thinkphp的模板漏洞之后server stopped，但是并没有给我弹回seterperter会话，我的猜测可能是之前太卡，我新开的命令端口，把前面的进程杀掉了。所以没有返回会话。

  4. 之后我在想复现到最后一个扫描端口的地方总是卡住，不知道什么原因世界未解之迷，建议官方整改

惊讶小秘密，这个靶机2还有个弱密码漏洞，123456.笑死
